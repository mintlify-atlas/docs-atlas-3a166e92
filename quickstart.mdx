## Prerequisites

Before starting, verify you have these tools installed:

| Tool | Minimum version | Check command |
|------|----------------|---------------|
| Docker | 24+ | `docker --version` |
| Docker Compose | 2.20+ | `docker compose version` |
| Git | Any recent | `git --version` |

<Warning>
  If you don't have Docker installed, visit [docker.com](https://www.docker.com/get-started) to download Docker Desktop for your operating system.
</Warning>

## Launch the system

<Steps>
  <Step title="Clone the repository">
    Open your terminal and clone the project:

    ```bash
    git clone <repository-url>
    cd calzado-jyr
    ```
  </Step>

  <Step title="Configure environment variables">
    Copy the example environment file and update it with your credentials:

    ```bash
    cp .env.example .env
    ```

    Edit `.env` and update these **required** fields:

    <CodeGroup>
    ```bash Database password
    POSTGRES_PASSWORD=your_secure_password_here
    ```

    ```bash JWT secret key
    # Generate a secure key:
    python -c "import secrets; print(secrets.token_urlsafe(48))"
    
    # Then paste it:
    SECRET_KEY=your_generated_secret_key
    ```

    ```bash Email configuration
    MAIL_USERNAME=noreply@calzadojyr.com
    MAIL_PASSWORD=your_smtp_app_password
    ```
    </CodeGroup>

    <Note>
      For Gmail SMTP, you need to create an [App Password](https://support.google.com/accounts/answer/185833) instead of using your regular password.
    </Note>
  </Step>

  <Step title="Start all services">
    Build and launch the complete stack (database, backend, frontend):

    ```bash
    docker compose up -d --build
    ```

    This command:
    - Builds Docker images for the backend and frontend
    - Starts PostgreSQL, FastAPI, and React services
    - Runs database initialization scripts
    - Enables hot-reload for development

    Verify all services are running:

    ```bash
    docker compose ps
    ```

    You should see three services in "Up" state: `db`, `be`, and `fe`.
  </Step>

  <Step title="Create the administrator account">
    Run the admin creation script inside the backend container:

    ```bash
    docker compose exec be python scripts/create_admin.py
    ```

    The script outputs the default credentials:

    ```
    ✅ Administrador creado exitosamente:
       Email: admin@calzadojyr.com
       Contraseña: Admin123!
       ⚠️  CAMBIA ESTA CONTRASEÑA después del primer login.
    ```

    <Warning>
      Change the default admin password immediately after your first login for security.
    </Warning>
  </Step>

  <Step title="Access the application">
    Open your browser and navigate to:

    | Service | URL | Description |
    |---------|-----|-------------|
    | **Frontend** | http://localhost:5173 | Main application interface |
    | **API Docs** | http://localhost:8000/docs | Interactive Swagger UI |
    | **Backend** | http://localhost:8000 | REST API endpoints |

    Login with the admin credentials to access the full system.
  </Step>
</Steps>

## First meaningful actions

<AccordionGroup>
  <Accordion title="Validate a customer account">
    After customers register through the public form, administrators must validate them:

    1. Login as admin at http://localhost:5173
    2. Navigate to **Users** → **Pending Customers**
    3. Review customer details
    4. Click **Validate** to activate the account
    5. Customer can now login and place orders
  </Accordion>

  <Accordion title="Create an employee account">
    Employees cannot self-register—only admins can create employee accounts:

    1. Login as admin
    2. Navigate to **Users** → **Create Employee**
    3. Fill in employee details:
       - Full name
       - Email address
       - Phone number
       - Occupation (Guarnición, Solador, Cortador, Emplantillador)
    4. Click **Create Employee**
    5. System generates temporary credentials and emails them to the employee
    6. Employee receives email with login credentials
    7. On first login, employee must change the temporary password
  </Accordion>

  <Accordion title="Test the API">
    Explore the interactive API documentation:

    1. Open http://localhost:8000/docs
    2. Click **POST /api/v1/auth/register** to expand the endpoint
    3. Click **Try it out**
    4. Enter sample customer data:
       ```json
       {
         "email": "test@example.com",
         "password": "SecurePass123!",
         "full_name": "Test Customer",
         "phone": "+57 300 1234567",
         "business_name": "Test Business"
       }
       ```
    5. Click **Execute** to register a test customer
    6. Check the response to see the created user (with `is_validated: false`)
  </Accordion>
</AccordionGroup>

## View logs

Monitor application activity in real-time:

<CodeGroup>
```bash All services
docker compose logs -f
```

```bash Backend only
docker compose logs -f be
```

```bash Frontend only
docker compose logs -f fe
```

```bash Database only
docker compose logs -f db
```
</CodeGroup>

## Stop the system

<CodeGroup>
```bash Stop services (preserve data)
docker compose down
```

```bash Stop and delete all data (complete reset)
docker compose down -v
```
</CodeGroup>

<Warning>
  The `-v` flag deletes the PostgreSQL volume, removing all users, orders, and data. Only use for development resets.
</Warning>
