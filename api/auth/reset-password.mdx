---
title: Reset password
description: Reset a user's password using a valid recovery token received via email.
---

## Endpoint

```
POST /api/v1/auth/reset-password
```

## Request body

<ParamField body="token" type="string" required>
  Password reset token received via email from the forgot password flow.
</ParamField>

<ParamField body="new_password" type="string" required>
  New password for the account. Must meet the following requirements:
  - Minimum 8 characters
  - At least one uppercase letter
  - At least one lowercase letter
  - At least one number
</ParamField>

## Response

<ResponseField name="message" type="string">
  Success message confirming the password reset.
</ResponseField>

## Status codes

<ResponseField name="200" type="OK">
  Password successfully reset.
</ResponseField>

<ResponseField name="400" type="Bad Request">
  Invalid or expired token, or new password does not meet requirements.
</ResponseField>

<ResponseField name="422" type="Unprocessable Entity">
  Validation error in request data.
</ResponseField>

## Example request

```bash
curl -X POST https://api.calzadojr.com/api/v1/auth/reset-password \
  -H "Content-Type: application/json" \
  -d '{
    "token": "abc123def456ghi789jkl012mno345pqr678",
    "new_password": "NewSecurePass456"
  }'
```

## Example response

```json
{
  "message": "Contraseña restablecida exitosamente"
}
```

## Error responses

### Invalid or expired token

```json
{
  "detail": "Token de recuperación inválido o expirado"
}
```

### Weak new password

```json
{
  "detail": [
    {
      "loc": ["body", "new_password"],
      "msg": "La contraseña debe tener al menos 8 caracteres",
      "type": "value_error"
    }
  ]
}
```

## Notes

- Reset tokens are single-use and expire after a set period (typically 1-24 hours)
- Once a token is used successfully, it cannot be reused
- The new password must meet the same strength requirements as during registration
- After a successful password reset, any existing access and refresh tokens for that user are invalidated
- Users should log in with their new password using the `/api/v1/auth/login` endpoint
