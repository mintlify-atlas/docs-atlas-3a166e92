---
title: Refresh token
description: Generate new access and refresh tokens using a valid refresh token.
---

## Endpoint

```http
POST /api/v1/auth/refresh
```

## Request body

<ParamField body="refresh_token" type="string" required>
  Valid refresh token obtained from the login endpoint.
</ParamField>

## Response

<ResponseField name="access_token" type="string">
  New JWT access token for authenticating API requests.
</ResponseField>

<ResponseField name="refresh_token" type="string">
  New JWT refresh token for future token refresh operations.
</ResponseField>

<ResponseField name="token_type" type="string">
  Type of token. Always returns `"bearer"`.
</ResponseField>

## Status codes

<ResponseField name="200" type="OK">
  Tokens successfully refreshed.
</ResponseField>

<ResponseField name="400" type="Bad Request">
  Invalid request format.
</ResponseField>

<ResponseField name="401" type="Unauthorized">
  Invalid or expired refresh token.
</ResponseField>

<ResponseField name="422" type="Unprocessable Entity">
  Validation error in request data.
</ResponseField>

## Example request

```bash
curl -X POST https://api.calzadojr.com/api/v1/auth/refresh \
  -H "Content-Type: application/json" \
  -d '{
    "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiI1NTBlODQwMC1lMjliLTQxZDQtYTcxNi00NDY2NTU0NDAwMDAiLCJleHAiOjE3MTAwNzYyMDB9.xyz789"
  }'
```

## Error responses

### Invalid or expired refresh token

```json
{
  "detail": "Token inválido o expirado"
}
```

## Notes

- Use this endpoint to obtain new tokens before the access token expires
- Both the access token and refresh token are rotated on each refresh for enhanced security
- If the refresh token is invalid or expired, the user must log in again
- Implement token refresh logic on the client side to maintain user sessions seamlessly
