---
title: Forgot password
description: Request a password recovery email with a reset token.
---

## Endpoint

```
POST /api/v1/auth/forgot-password
```

## Request body

<ParamField body="email" type="string" required>
  Email address of the account to recover. Must be a valid email format.
</ParamField>

## Response

<ResponseField name="message" type="string">
  Generic success message for security purposes (sent regardless of whether the email exists).
</ResponseField>

## Status codes

<ResponseField name="200" type="OK">
  Request processed successfully.
</ResponseField>

<ResponseField name="400" type="Bad Request">
  Invalid request format.
</ResponseField>

<ResponseField name="422" type="Unprocessable Entity">
  Validation error (e.g., invalid email format).
</ResponseField>

## Example request

```bash
curl -X POST https://api.calzadojr.com/api/v1/auth/forgot-password \
  -H "Content-Type: application/json" \
  -d '{
    "email": "cliente@example.com"
  }'
```

## Example response

```json
{
  "message": "Si el email está registrado, recibirás un enlace de recuperación"
}
```

## Notes

- For security reasons, the API returns the same response whether the email exists or not
- If the email is registered, the user will receive an email with a password reset link
- The reset token in the email is time-limited and single-use
- The reset link typically follows the format: `https://app.calzadojr.com/reset-password?token={reset_token}`
- Use the token received via email with the `/api/v1/auth/reset-password` endpoint to complete the password reset
