---
title: Change password
description: Change the password for an authenticated user.
---

## Endpoint

```
POST /api/v1/auth/change-password
```

## Authentication

This endpoint requires authentication. Include a valid access token in the request header:

```
Authorization: Bearer {access_token}
```

## Request body

<ParamField body="current_password" type="string" required>
  User's current password for verification.
</ParamField>

<ParamField body="new_password" type="string" required>
  New password. Must meet the following requirements:
  - Minimum 8 characters
  - At least one uppercase letter
  - At least one lowercase letter
  - At least one number
</ParamField>

## Response

<ResponseField name="message" type="string">
  Success message confirming the password change.
</ResponseField>

## Status codes

<ResponseField name="200" type="OK">
  Password successfully changed.
</ResponseField>

<ResponseField name="400" type="Bad Request">
  Invalid request data or new password does not meet requirements.
</ResponseField>

<ResponseField name="401" type="Unauthorized">
  Missing or invalid access token, or incorrect current password.
</ResponseField>

<ResponseField name="422" type="Unprocessable Entity">
  Validation error in request data.
</ResponseField>

## Example request

```bash
curl -X POST https://api.calzadojr.com/api/v1/auth/change-password \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  -d '{
    "current_password": "SecurePass123",
    "new_password": "NewSecurePass456"
  }'
```

## Example response

```json
{
  "message": "Contraseña actualizada exitosamente"
}
```

## Error responses

### Incorrect current password

```json
{
  "detail": "La contraseña actual es incorrecta"
}
```

### Weak new password

```json
{
  "detail": [
    {
      "loc": ["body", "new_password"],
      "msg": "La contraseña debe contener al menos una letra mayúscula",
      "type": "value_error"
    }
  ]
}
```

### Missing or invalid token

```json
{
  "detail": "No autenticado"
}
```

## Notes

- The user must provide their current password to change it
- The new password must meet the same strength requirements as during registration
- After a successful password change, existing tokens remain valid until expiration
- The password is securely hashed before storage
